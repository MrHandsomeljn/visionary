## Feature 9：右侧面板排版与表单交互细节

### 实现内容

- 标签与输入框对齐，三行（平移/旋转/缩放）结构规整。

## Feature 18：时间轴 UI 对齐与交互精度细节（相机模块收尾）

### 实现内容

- 时间值显示从滑块区迁移到标题栏右侧（FPS 后），释放滑块整段宽度。
- 滑块轨道底色按需求移除，避免影响刻度阅读。
- 关键帧区高度按反馈调整到 `20px`，标尺下方区域作为有效关键帧显示区。
- 关键帧点大小、占位文案、垂直居中关系统一，提升可读性与点击体验。

### 过程问题

- 问题A：滑块看似与标尺不对齐，端点在圆球边缘。
  - 原因A：视觉对齐基于拇指边缘，而用户需要圆心对齐刻度。
  - 解决A：刻度/关键帧/光标/点击选点统一改为“圆心可达区间映射”。
- 问题B：标尺和关键帧区高度关系不合理，DevTools 观感为“标尺吃掉下方空间”。
  - 原因B：标尺高度与基线位置设计偏上，关键帧区又被弹性拉伸。
  - 解决B：下调标尺高度、基线贴底、关键帧区固定高度，整体改为内容自适应贴底布局。
- 问题C：不同 FPS 下拖不到 10 秒整点。
  - 原因C：`1/fps` 浮点步长截断导致累计误差。
  - 解决C：滑块改为帧整数域，时间文本由帧换算。

### 验证结果

- 时间轴交互在视觉与数值上达成一致：  
  可准确到端点、可准确对齐刻度、可稳定显示与编辑关键帧。

## Feature 19：导出弹窗交互与参数同步细节

### 实现内容

- 新增导出弹窗 UI（分辨率/渲染模式/FOV），并支持“取消/渲染”交互。
- 分辨率默认值自动取当前视窗尺寸（并自动补齐常用预设 720p/1080p/2K/4K）。
- 导出模式默认值对齐当前编辑器模式；FOV 默认值对齐当前场景 FOV。
- 视频导出弹窗内增加时间轴摘要提示（时长、FPS、总帧数、关键帧数量）。

### 过程问题

- 问题A：原顶部“渲染”按钮是模式切换行为，不符合“一次性导出动作”。
  - 原因A：之前把“导出入口”做成了状态按钮。
  - 解决A：移除状态切换逻辑，改为按钮只负责打开弹窗并执行单次导出任务。
- 问题B：导出时临时切换渲染模式会污染当前编辑器状态。
  - 原因B：导出参数与预览参数共用同一渲染后端。
  - 解决B：增加“临时渲染模式包裹”逻辑，导出前切换、导出后自动恢复原模式。
- 问题C：视频导出过程中需要避免 UI 操作打断。
  - 原因C：导出是异步长任务，重复点击会造成竞态。
  - 解决C：增加导出中锁定（禁用表单/按钮、遮罩状态与加载提示）。

### 验证结果

- 导出交互已经满足“一次性动作 + 参数弹窗 + 与当前编辑器状态一致默认值”。

## Feature 20：时间轴相机在 3D 场景内可视化（0.0.9 细节）

### 实现内容

- `EditorApp` 新增 `setCameraSequenceVisualization()` / `clearCameraSequenceVisualization()`，接收时间轴关键帧并生成 3D helper。
- 每个关键帧生成一组 `LineSegments` 锥体线框：
  - 顶点使用关键帧相机中心；
  - 底面四角根据关键帧 `fov` 与场景 `aspect` 计算；
  - 朝向使用关键帧四元数反算到相机前向。
- 新增轨迹线：将关键帧相机中心按时间排序后连线。
- 新增当前相机 marker：每帧渲染循环中由当前 mesh 相机位置驱动。
- `public/editor.js` 增加 `syncCameraSequenceVisualization()`，并在关键帧结构变化处统一调用。

### 过程问题

- 问题A：编辑器与导出模块使用场景不同，不能直接把 `RecordingCamera` 整个实例搬进编辑器主循环。
  - 原因A：`RecordingCamera` 绑定离屏渲染/导出流程，生命周期和依赖比编辑器 helper 重。
  - 解决A：只复用其几何语义与坐标约定，编辑器侧实现轻量 helper，避免引入导出链路耦合。
- 问题B：helper 若不做资源回收，频繁改关键帧会堆积 geometry/material。
  - 原因B：关键帧每次改动都要重建可视化对象。
  - 解决B：新增 `disposeObject3DResources()`，在重建与 `dispose()` 时显式回收。
- 问题C：helper 可能污染后续流程（如被误认为业务模型）。
  - 原因C：编辑器场景对象与业务对象共用同一 `meshScene`。
  - 解决C：统一给 helper 打 `__visionaryEditorHelper` 标记并集中放入 `cameraSequenceGroup` 管理。

### 验证结果

- 时间轴关键帧在场景内可见，轨迹与当前相机 marker 能同步更新，且不会影响模型加载/删除主流程。

## Feature 21：相机可视化与播放链路细节收尾（0.0.9 增量）

### 实现内容

- 相机锥体边与轨迹线从 `LineSegments/Line` 改为“圆柱段”厚线几何，避免 WebGPU 下线宽不可控。

- 时间轴播放引入轻量 UI 刷新路径：
  - `setTimelineFrame(..., { lightweightUi: true })`
  - `updateTimelineCursorOnly()` 只更新游标、时间文本、关键帧选中态与删除按钮禁用状态。
- `EditorApp` 增加相机交互回调 `onCameraInteraction`，播放中检测到拖拽/滚轮/键盘输入时自动暂停。

## Feature 22：UI 面板滚动优化（0.0.9 细节）

### 实现内容

- 改写全局级别滚动条 `::-webkit-scrollbar`，隐藏原生厚重 UI。
- 设计了圆角和半透明轨道的拇指 (`thumb`)，使面板在大量参数折叠与展开时拥有平滑且不阻挡视线的滚动体验。
- 修复了弹性布局下（`display: flex`）子元素默认收缩导致的滚动条失效问题。
- 轨迹线生成方式改为“时间轴插值函数采样”，由 `buildSampledCameraTrajectory()` 产出并传给 `setCameraSequenceVisualization`。

### 过程问题

- 问题A：相机辅助体会被 PLY 遮挡，同时又覆盖 GLB，遮挡行为分裂。
  - 原因A：辅助体材质使用 `depthTest=false/depthWrite=false` 且高 `renderOrder`。
  - 解决A：改为参与深度测试与写入，并取消强制高层级渲染顺序。
- 问题B：新增关键帧后锥体“越加越粗”。
  - 原因B：粗细依赖关键帧空间跨度，关键帧范围变化会放大半径。
  - 解决B：改为基于场景模型尺度估算锥体长度/线半径，移除和关键帧数量的耦合。
- 问题C：轨迹线看起来与播放路线不一致。
  - 原因C：可视化轨迹是关键帧折线，而播放路径是插值轨迹。
  - 解决C：轨迹改为按播放同源插值函数采样，保证视觉轨迹与实际播放路径一致。
- 问题D：播放卡顿明显高于手动拖动。
  - 原因D：播放每帧重绘时间轴标尺与关键帧 DOM，导致主线程开销过大。
  - 解决D：播放时启用轻量 UI 刷新，并在同帧情况下仅更新时间文本，避免冗余重绘。
- 问题E：用户播放过程中手动调整相机，仍被时间轴下一帧强制拉回。
  - 原因E：缺少“手动相机输入 -> 退出播放”联动机制。
  - 解决E：在 `EditorApp` 相机输入路径触发交互回调，前端收到后立即 `stopTimelinePlayback(false)`。

### 验证结果

- 相机可视化链路已闭环：  
  粗细稳定、遮挡正确、导出不污染、轨迹与播放一致；  
  播放中可手动接管相机且会自动暂停，时间轴播放流畅度明显改善。

## Feature 24：动画速率输入框实时焦点保留修正（0.0.9 细节）

### 实现内容

- 在 `public/editor.js` 的 `updateModelAnimationControls(id)` 函数内，对 `dom.modelAnimSpeedNumber` 数值回填逻辑新增了 `document.activeElement !== dom.modelAnimSpeedNumber` 焦点保护判定。
- 确保时间轴播放、模型选中的高频 UI 属性刷新循环中，如果用户正在编辑文本框（例如敲击 `2.` 未完成时），不会触发 `.value =  speed.toFixed(3)` 的强制覆写断点。

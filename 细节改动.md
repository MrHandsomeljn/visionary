# Visionary Editor 工作报告（交互 Feature 版）

## Feature 9：右侧面板排版与表单交互细节

### 实现内容

- 标签与输入框对齐，三行（平移/旋转/缩放）结构规整。
- 缩放输入横向占位优化，减少截断。
- 去掉旋转标签的小圆圈符号。
- 去掉输入框上下箭头，输入空间更大。
- 标签拖拽改值（如 X/Y/Z 左右拖动）。

### 过程问题

- 问题：输入框显示不完整、排版松散、交互效率低。
- 原因：控件宽度与布局网格不统一，默认 number input UI 冗余。
- 解决：重排表单布局、统一宽度/对齐规则、改成更紧凑输入样式并加拖拽调参。

### 验证结果

- 你多轮反馈后确认可用，并继续用于细节迭代。

## Feature 10：模型列表与按钮文案细节

### 实现内容

- 删除“加载模型”，保留单一“添加模型”入口，避免重复路径。
- 可见性控制从右侧面板迁到模型列表“可见”按钮。
- 按钮文本从单字改为词语（如“播->播放”“环->循环”）。
- 菜单项“保存/加载/清空”改纯文字，去掉 icon。

### 过程问题

- 问题：按钮语义不统一、单字缩写歧义大、入口重复。
- 原因：早期快速实现，未做信息架构收敛。
- 解决：统一词语化命名，收敛功能入口，列表侧承担模型状态操作。

### 验证结果

- 你对文案与入口调整持续给反馈并确认方向。

## Feature 11：场景/模型设置区块化与显隐策略

### 实现内容

- 场景设置与模型设置拆成两张卡片。
- 标题层级增强，子项缩进回调（减少过度缩进）。
- 未选中模型时：模型变换区灰化，ONNX 动画区隐藏。

### 过程问题

- 问题：场景项混在模型项里，信息层级不清。
- 原因：模块边界最初未拆分。
- 解决：按“场景级 vs 模型级”重构面板结构与状态显示规则。

### 验证结果

- 你明确提出该结构需求并确认已按方向实现。

## Feature 12：主题与视觉细节

### 实现内容

- 明/暗主题切换入口放到右上角版本号旁。
- 默认主题固定为白天模式。
- 深度模式背景观感优化（避免整屏硬黑）。

### 过程问题

- 问题：默认主题与预期不一致、深度模式背景可读性差。
- 原因：初始配色/模式切换策略未固化。
- 解决：明确默认白天；深度无几何像素回退场景底色。

### 验证结果

- 你确认默认白天；深度模式观感进入可用状态。

## Feature 13：数值体系细节与一致性

### 实现内容

- 控件数值精度统一到小数点后三位。
- 深度倍率支持更小值（如 0.01）。
- 深度倍率修改可实时作用到渲染。

### 过程问题

- 问题：不同控件精度不一致、深度倍率早期不生效。
- 原因：输入层与渲染层参数同步链路不完整。
- 解决：统一输入格式化策略，并把倍率同步到实际渲染参数更新路径。

### 验证结果

- 你反馈后多轮迭代，最终深度控制进入正常可用。

## Feature 14：空状态与显示文案细节

### 实现内容

- 模型列表空状态文案清理与初始化闪现问题排查。
- UI 文本去 emoji 化，减少非必要符号噪声。

### 过程问题

- 问题：页面初期出现“空白”闪现，文案风格不统一。
- 原因：初始 DOM 模板与运行时渲染存在短暂切换。
- 解决：清理静态/动态空状态来源并统一文案策略，仓库规则补充“UI 尽量少 emoji”。

### 验证结果

- 你对该类细节持续追踪并纳入稳定版本提交。

### 验证结果

## Feature 16：时间轴刻度与帧率交互细节

### 实现内容

- 底部时间轴右侧新增 FPS 下拉（12/24/30/60）。
- 时间轴标尺按帧绘制：每秒主刻度标注 `Ns`，每帧细刻度可见。
- 时间轴轨道支持点击选中时间戳，滑块输入会自动吸附到最近帧。
- 关键帧点位在轨道内可视化，点击点位可直接选中对应时间戳。

### 过程问题

- 问题：原有 slider 是连续时间，不具备帧语义，无法做到“在某个小格上新增/覆盖关键帧”。
- 原因：缺少统一的 `time <-> frame` 映射与帧吸附机制。
- 解决：引入 `timeToFrame/frameToTime` 映射、统一帧级存储、slider 步长改为 `1/fps`、点击选中统一走帧编号。

### 验证结果

- 时间轴交互从“占位显示”变为可操作状态，具备帧级关键帧编辑能力。

## Feature 17：场景相机 FOV 控件细节

### 实现内容

- 场景设置新增 FOV 滑条与数字输入框，精度统一到 3 位小数。
- FOV 参数与渲染相机立即同步，无需刷新或重载模型。
- 关键帧中记录 FOV，并在时间轴插值/播放时同步恢复视角参数。

### 过程问题

- 问题：原场景设置只有背景和深度参数，缺少相机光学参数入口。
- 原因：编辑器 UI 与相机投影视锥参数未打通。
- 解决：补齐 UI 输入、状态同步、`EditorApp -> CameraManager` FOV API，以及关键帧对 FOV 的序列化路径。

### 验证结果

- FOV 控件已进入可测试状态，可作为相机序列参数的一部分联动时间轴。

## Feature 18：时间轴 UI 对齐与交互精度细节（相机模块收尾）

### 实现内容

- 时间值显示从滑块区迁移到标题栏右侧（FPS 后），释放滑块整段宽度。
- 滑块轨道底色按需求移除，避免影响刻度阅读。
- 关键帧区高度按反馈调整到 `20px`，标尺下方区域作为有效关键帧显示区。
- 关键帧点大小、占位文案、垂直居中关系统一，提升可读性与点击体验。

### 过程问题

- 问题A：滑块看似与标尺不对齐，端点在圆球边缘。
  - 原因A：视觉对齐基于拇指边缘，而用户需要圆心对齐刻度。
  - 解决A：刻度/关键帧/光标/点击选点统一改为“圆心可达区间映射”。
- 问题B：标尺和关键帧区高度关系不合理，DevTools 观感为“标尺吃掉下方空间”。
  - 原因B：标尺高度与基线位置设计偏上，关键帧区又被弹性拉伸。
  - 解决B：下调标尺高度、基线贴底、关键帧区固定高度，整体改为内容自适应贴底布局。
- 问题C：不同 FPS 下拖不到 10 秒整点。
  - 原因C：`1/fps` 浮点步长截断导致累计误差。
  - 解决C：滑块改为帧整数域，时间文本由帧换算。

### 验证结果

- 时间轴交互在视觉与数值上达成一致：  
  可准确到端点、可准确对齐刻度、可稳定显示与编辑关键帧。

# 0.1.8

## Added

- Added writable workspace persistence for editor scene state, workspace snapshot orchestration, and save-status feedback.
- Added workspace-aware agent conversation archival with `agent_history.json` plus hashed binary assets in `agent_history/`.
- Added local-development support for trusted HTTPS, LAN-accessible startup, and post-install guidance for new machines.
- Added regression coverage for workspace dirty hooks, workspace indicator behavior, camera preview sizing/positioning policy, i18n wiring, and agent session storage.

## Changed

- Changed the editor save flow from demo-oriented temporary state toward an actual workspace editing workflow with staged sync semantics.
- Changed the workspace indicator to aggregate scene changes and agent-history changes into one unified dirty/saving/error state.
- Changed camera preview behavior so open state and window position are restored per workspace, while preview size falls back to the default on page refresh.
- Changed camera preview first-open placement to anchor beside the agent workbench and above the timeline, using measured panel geometry instead of cached browser layout assumptions.
- Changed several editor controls and text wiring to be more consistent with declarative i18n and current theme behavior.

## Fixed

- Fixed multiple missing dirty-state hooks, including timeline camera FOV edits, scene camera FOV edits, scene background changes, visibility changes, and transform no-op cases.
- Fixed workspace save behavior so agent history participates in persistence and status reporting instead of remaining outside the workspace lifecycle.
- Fixed camera preview preset restoration bugs caused by hidden-state sizing and stale browser-level size caching.
- Fixed editor development setup drift by documenting and scripting the HTTPS/LAN path required for cross-device access.

## Notes

- This version closes the shift from demo playback tooling toward persistent local workspace editing.
- The main outcome of `0.1.8` is that workspace state, agent state, save feedback, and camera preview panel behavior are now maintained as one coherent editor workflow.

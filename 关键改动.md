# Visionary Editor 工作报告（核心 Feature 版）

## Feature 1：编辑器可运行架构（模块链路重建）

### 实现内容

- 将编辑器核心逻辑迁入 src/editor/editor-app.ts，恢复 Vite 对 TS 与裸模块的解析能力。
- 保留前端页面入口，但实际能力由 src 模块驱动。

### 过程问题

- 问题：Uncaught TypeError: Failed to resolve module specifier "three/webgpu"。
  - 原因：public/ 下脚本不经过 Vite 模块解析流程。
  - 解决：改为 src 模块化入口，three/webgpu、onnxruntime-web、管理器模块均走构建链路。

### 验证结果

- 拖放可触发、模型可加载

## Feature 2：统一融合渲染管线， 3DGS + Mesh 同场景

### 实现内容

- 建立单循环渲染：相机更新 -> Three 场景渲染到 RT -> 深度/颜色复用 -> Gaussian 叠加。
- 支持 ply/onnx 与 glb/gltf/obj/fbx/stl 共存显示。
- 删除模型时同步从管理层、Three 场景、融合渲染器三处剔除。

### 过程问题

- 问题：出现“列表删除了但画面残留”“一度全黑/空画面”。
  - 原因：模型生命周期与融合渲染状态不同步，且部分分支 fallback 覆盖了目标输出。
  - 解决：统一删除路径与渲染顺序，补齐模型移除与回收逻辑，修正 fallback 条件。

### 验证结果

- 统一渲染恢复，方向一致，删除行为已正常

## Feature 3：Mesh 导入能力（从无到有）

### 实现内容

- 新增 Mesh 文件识别与加载分支（glb/gltf/obj/fbx/stl）。
- Mesh 对象进入同一场景、同一相机、同一渲染模式切换系统。

### 过程问题

- 问题：KHR_materials_pbrSpecularGlossiness 扩展导致 glTF 警告/显示异常。
  - 原因：扩展兼容不完整。
  - 解决：在 threejs_adapters.ts 增加材质兜底策略，保证可见性与稳定显示。

### 验证结果

- glb 现在可显示。

## Feature 4：深度图/法向图渲染（重点）

### 实现内容

- 编辑器模式切换可用：color / normal / depth。
- 法向图稳定可用。
- 深度图支持 ply + glb，并与倍率参数联动。

### 过程问题

- 问题A：mesh 深度全黑。
  - 原因A：three/webgpu 下 MeshDepthMaterial 不兼容。
  - 解决A：改为直接使用 sceneDepthTexture 的全屏深度可视化 pass。
- 问题B：深度线性化异常，整屏接近最远深度。
  - 原因B：分母钳制符号错误（把应保持负值的分母错误钳制）。
  - 解决B：将分母钳制逻辑修正为保持负号区间，恢复正确 viewZ。
- 问题C：天空在深度模式下纯黑。
  - 原因C：无几何像素也按深度映射输出。
  - 解决C：对 depth≈1 像素回退 RT 场景颜色。
- 问题D：ply 与 glb 深度层次不一致，且模型缩放会影响映射。
  - 原因D：两条路径映射基准不同，且基准使用了实时 world 尺寸。
  - 解决D：统一参考尺度逻辑，并把基准改为“加载时固定尺寸/局部尺度”，将深度倍率与模型缩放解耦。

### 验证结果

- 深度渲染功能已经正常，ply 不再跟着 mesh 变化。

## Feature 5：相机与交互控制系统

### 实现内容

- 鼠标与键盘相机控制完整接入（WASD/QE/R/Shift+R/PgUp/PgDn）。
- 快捷键：F 聚焦选中模型，X 回正视图。
- 固定当前控制手感方案并保持不回退。

### 过程问题

- 问题：A/D 与鼠标拖动方向多轮不一致；俯视接近 90° 时出现旋转异常。
  - 原因：输入映射符号与坐标约定不一致，极角附近姿态稳定性不足。
  - 解决：统一 mouse delta 与键位 remap 逻辑，补齐回正/聚焦时上方向约束。

### 验证结果

- 你明确反馈“现在操控是对的，请永远保持”。

## Feature 6：模型管理与属性编辑（可操作化）

### 实现内容

- 左侧模型列表支持新增、删除、可见开关、选中联动。
- 右侧属性编辑支持位置/旋转/缩放实时改值、重置。
- 属性输入精度统一到 3 位小数。
- 标签拖拽改值能力接入（不依赖输入框小箭头）。

### 过程问题

  - 原因：UI 状态与底层模型对象未双向绑定。
  - 解决：统一以模型记录驱动 ModelEntry + Object3D + GaussianModel 三层同步。

### 验证结果

## Feature 9：相机序列时间轴（关键帧系统）与场景 FOV 控制

### 实现内容

- 场景设置新增 `FOV` 控件（滑条 + 数值框），修改后立即作用到渲染相机。
- 时间轴新增 `FPS` 选项，并将时间轴量化到帧（`1s` 大格，`1/fps` 小格）。
- 时间轴支持“按时间戳选帧”，并基于选中帧执行关键帧“新增/覆盖/删除”。
- 关键帧存储内容为完整相机位姿（position + rotation + fov），并支持插值预览与播放。

### 过程问题

- 问题：原时间轴逻辑是占位实现，只有按钮与 slider，没有真实帧率、刻度、关键帧数据结构与相机驱动。
  - 原因：早期 UI 完成但未接入可执行的数据模型与渲染相机接口。
  - 解决：补齐 `EditorApp`/`CameraManager` 的相机姿态与 FOV 读写 API；在前端重建帧级时间轴状态机、刻度渲染、关键帧增删覆盖与相机应用路径。

### 验证结果

- 功能已可进入手工测试：FOV 即时生效，时间轴可选帧并写入/删除关键帧，FPS 与刻度尺联动。

## Feature 10：相机序列模块收尾（可用性与精度闭环）

### 实现内容

- 时间轴布局从“占位高度”改为“内容自适应 + 贴浏览器底边”，去除关键帧模块下方空白区。
- 相机关键帧区域、标尺、滑块三者对齐规则统一，确保时间选择与视觉刻度一致。
- 时间滑块从“秒浮点步长”改为“帧整数步长”（`0..totalFrames`），显示时间由 `frame/fps` 计算。
- 去除“至少 2 个关键帧才能播放”限制，允许直接进入播放状态机（按当前数据运行）。

### 过程问题

- 问题A：时间轴滑块在 `fps=24/60` 时无法拖到 `10.000s`，只能到 `9.958s/9.983s`。
  - 原因A：`step` 使用浮点并截断精度后累计误差，最后一帧超出 `max` 被浏览器裁剪。
  - 解决A：改为帧域滑块（整数步长），彻底消除浮点步长累积误差。
- 问题B：滑块圆心与标尺/关键帧点不重合，端点对齐在圆球边缘而非圆心。
  - 原因B：位置映射按容器宽度线性插值，未扣除拇指直径可达区间。
  - 解决B：统一采用“半拇指偏移 + 有效宽度映射”公式，点击选帧与渲染点位共用同一映射。
- 问题C：关键帧模块高度与内部空间分配不合理，出现“标尺占位过高”和下方空白。
  - 原因C：容器固定高 + 内部 `flex:1` 拉伸导致实际可视区域与功能区不匹配。
  - 解决C：重设标尺/关键帧区高度，并将底部模块改为 `auto` 高度贴底。

### 验证结果

- 相机关键帧模块已进入可发布状态：  
  关键帧编辑、帧率刻度、时间端点、视觉对齐和底部布局均完成闭环。

## Feature 11：渲染导出工作流（图片/视频）接入官方导出管线

### 实现内容

- 顶部“渲染”菜单改为一次性动作按钮：点击“视频/图片”直接打开导出弹窗，不再保存按钮激活状态。
- 新增统一导出弹窗，支持三项参数：分辨率、渲染模式（图片/深度图/法向图）、FOV。
- 图片导出接入官方 `RecordingCamera` 渲染链路，按弹窗参数离屏渲染后下载 PNG。
- 视频导出接入官方 `exportVideoWithRecordingCamera`，并将编辑器时间轴关键帧桥接为 `ITimelineController`，实现“按时间轴逐帧导出视频”。

### 过程问题

- 问题A：官方 `exportImage` 仅导出当前主画布，无法满足“自定义分辨率/FOV/渲染模式”。
  - 原因A：该接口是快捷截图接口，不包含离屏参数化渲染。
  - 解决A：改走官方 `RecordingCamera` 的离屏渲染路径，保持官方渲染栈，同时满足弹窗参数化导出需求。
- 问题B：视频导出需要时间轴驱动，但官方导出依赖 `ITimelineController` 抽象接口。
  - 原因B：编辑器内部时间轴状态机与 Core 导出接口未直接连通。
  - 解决B：实现编辑器侧 `ITimelineController` 适配器（总帧数/帧率/帧索引设置/帧更新回调），把现有关键帧系统映射到官方导出流程。
- 问题C：导出深度/法向时需要录制相机与主视图保持同一预览模式与深度倍率。
  - 原因C：`RecordingCamera` 内部独立的 `GaussianThreeJSRenderer` 默认处于 color 模式。
  - 解决C：为 `RecordingCamera` 增加预览模式与深度倍率同步接口，并在导出时显式下发，保证导出结果与编辑器配置一致。

### 验证结果

- 图片/视频导出链路已经打通，且视频导出可直接复用时间轴关键帧过程。
